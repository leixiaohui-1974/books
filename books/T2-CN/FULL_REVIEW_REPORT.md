## 五、图片生成问题诊断与修复方案

### 问题根因

`generate_images.py` 脚本存在根本性缺陷：**它从未调用任何 AI 图像生成 API**。

当前代码实际行为：
```python
# 实际执行的是这个——只创建纯色背景+文字的占位图
img = Image.new('RGB', (1024, 768), color=(240, 248, 255))  # 淡蓝色背景
draw = ImageDraw.Draw(img)
draw.text((512, 384), f"Fig: {key}", fill=(0, 0, 0), anchor="mm")  # 只写标签
img.save(output_file)  # 保存到 H/ 目录
```

此外，`generate_images.py` 中的提示词（英文简短描述）与 `chapter_image_prompts.md` 中精心设计的详细提示词**完全不同**，后者才是适合 Nano Banana 等工具的正式提示词。

### 修复方案对比

| 方案 | 适用场景 | 复杂度 | 效果 |
|------|---------|--------|------|
| **方案A：手动逐图生成** | 有 Nano Banana / Midjourney 账号 | 低 | 最佳效果 |
| **方案B：调用 DALL-E API** | 有 OpenAI API key | 中 | 良好 |
| **方案C：调用 Stable Diffusion 本地** | 本机有足够 GPU 内存 | 高 | 良好 |
| **方案D：高质量占位图替代** | 临时交付 | 低 | 可接受 |

**推荐方案A（手动 Nano Banana）：**

`chapter_image_prompts.md` 已包含 14 张图的完整英文提示词，格式专为 Nano Banana 设计。只需：
1. 打开 Nano Banana
2. 逐一粘贴提示词
3. 生成后下载 PNG，保存至 `H/fig_ch{N}_{M}.png`

**如需自动化方案（方案B）：** 可提供集成 OpenAI Image API 的修复版 `generate_images.py`，需要您提供 API key。
