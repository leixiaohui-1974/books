# 写作技法指南 (Writing Craft Guide)

> 本文件定义各文体的写作技法——不是"检查什么"，而是"怎么写好"。
> 写作引擎在撰写初稿(§4.1步骤6)时必须遵循本指南。

---

## 通用写作原则（所有文体适用）

### 一句话归零测试
每写完一段，自问：「删掉这段，文章是否受损？」如果不受损，删掉它。

### 信息密度三层法
- **第一层（骨架）**: 每段第一句承载核心信息，只读第一句就能理解全文脉络
- **第二层（肌肉）**: 展开论据、数据、推导
- **第三层（血液）**: 过渡、衔接、节奏调节

### 主动语态优先
- ✅ "We propose a distributed MPC framework"
- ❌ "A distributed MPC framework is proposed"
- ✅ "本文建立了IDZ模型参数解析映射关系"
- ❌ "IDZ模型参数解析映射关系被建立"

### 具体胜过抽象
- ❌ "性能显著提升"
- ✅ "水位偏差从±15cm降至±3cm，响应时间缩短62%"

## SCI英文论文写作技法

### Introduction写法（倒三角结构）

```
第1段: 大背景 — 为什么这个领域重要（2-3句，最宽）
第2段: 小背景 — 这个具体问题是什么（引文密集）
第3段: 差距(Gap) — 前人做了什么，还差什么（这是论文的心脏）
第4段: 本文贡献 — 我们做了什么来填补差距（3个contributions，编号列出）
第5段(可选): 论文路线图 — "The remainder of this paper is organized as..."
```

**Gap Statement句型库**：
- "However, {existing approaches} fail to account for {our concern}."
- "Despite significant progress in {X}, the {Y} remains poorly understood."
- "A critical limitation of {prior work} is that {specific limitation}."
- "To date, no systematic framework exists for {our topic}."
- "While {A} has been well studied, its extension to {B} is non-trivial because {reason}."

**Contribution Statement句型库**：
- "The main contributions of this paper are threefold: (1)... (2)... (3)..."
- "Specifically, we (i) establish..., (ii) develop..., and (iii) validate..."
- "To address this gap, we propose {method}, which {key advantage}."

### Methodology写法

**定理/引理结构**：
```
Definition X (术语). 定义正文...
Assumption X. 假设正文...（每个假设必须有物理解释）
Theorem X. 定理陈述...
Proof. 推导过程... □

Remark X. 对定理的物理直觉解释、适用范围讨论。
```

**关键技巧**：
- 每个公式后面跟一句"物理直觉"解释（"Physically, this means that..."）
- 复杂推导在正文保留关键步骤，详细推导放Appendix
- 参数表必须包含：符号、含义、单位、取值范围、来源

### Results写法

**图表解读三段式**：
```
句1(事实): "Fig. 3 shows that the IDZ model reproduces the step response within 2% error."
句2(对比): "Compared to the full Saint-Venant simulation, the computational time is reduced by 95%."
句3(洞察): "This suggests that IDZ is sufficient for real-time control applications where..."
```

**讨论的深度来源**：
- 为什么这个结果是这样的？（机理解释）
- 什么条件下这个结果不成立？（边界条件）
- 这个结果和别人的结果比怎么样？（文献对比）
- 这个结果的工程意义是什么？（应用推论）

### Conclusions写法
- 第1段：回顾问题+方法（2句）
- 第2段：三个核心发现（用past tense）
- 第3段：局限性（诚实且具体）
- 第4段：未来方向（要有新意，不是套话）

**禁忌**：不要在Conclusions中引入新数据或新论点。

## 中文核心论文写作技法

### 与SCI的关键差异
- **引言更重文献对比**：中文论文引言通常需要更详细的国内外研究综述表格
- **更重工程验证**：审稿人特别关注"在XX工程中验证"
- **摘要结构**：目的—方法—结果—结论四段式，每段1-2句

### 中文学术表达升级

| 低级表达 | 高级表达 |
|---------|---------|
| 本文研究了… | 针对…问题，提出了…方法 |
| 结果表明… | 数值实验/工程验证表明… |
| 有很大提高 | 较传统方法提升了X%（具体量化） |
| 进行了分析 | 从…三个维度开展了系统分析 |
| 具有重要意义 | 为…提供了理论依据/技术支撑 |

### 图表双语标题格式
```
图 3 南水北调中线某渠池水位阶跃响应
Fig. 3 Step response of water level in a canal pool of the Middle Route of SNWTP
```

## 发明专利写作技法

### 权利要求书撰写核心技法

**独立权利要求（前序+特征两段式）**：
```
1. 一种[技术主题]，包括[前序部分——已知技术特征]，
   其特征在于：[特征部分——创新技术特征，分点列出]：
   步骤一，[动作]，具体为[详述]；
   步骤二，[动作]，所述[术语]通过[方式]实现[效果]；
   ...
```

**从属权利要求层次设计**：
```
权1: 方法独权（最宽保护范围）
  ├─ 权2: 限定步骤一的具体算法
  ├─ 权3: 限定步骤二的参数范围
  ├─ 权4: 限定数据处理方式
  └─ 权5: 限定实施场景
权6: 系统独权（对应方法独权的系统实现）
  ├─ 权7: 限定模块一的具体结构
  ├─ 权8: 限定模块二的接口协议
  └─ 权9: 限定存储介质
权10: 计算机可读存储介质独权
```

**关键语言规范**：
- 用"所述"回指前面出现的技术特征（不用"该""上述"）
- 方法权利要求用动词开头（"获取…""计算…""判断…"）
- 不写"优选""最好"（这是说明书的事，不是权要的事）
- 数值范围用"X至Y"或"X-Y"，不用"大约""左右"

### 背景技术写法
```
段1: 技术领域背景（1-2句）
段2: 现有技术方案1及其缺陷
段3: 现有技术方案2及其缺陷
段4: 现有技术方案3及其缺陷（引用公开号）
段5: 小结——上述方案的共同不足（引出本发明要解决的问题）
```

### 具体实施方式写法
- 至少一个完整实施例，含**全部具体参数**（不能只写"合适的值"）
- 参数表：控制周期、采样间隔、预测步长、约束范围等全部量化
- 从最宽的实施方式开始，逐步具体化
- 数据图：输入→处理→输出的完整链条

## 书稿/教材写作技法

### 章首导言写法（300-500字）
```
段1: 提出一个引发好奇心的问题/场景
段2: 本章要解决什么问题（目标）
段3: 本章的知识路径图（从X出发，经过Y，到达Z）
段4: 本章与前后章的关系（"在第2章介绍了...的基础上，本章将..."）
```

### 概念引入三步法
```
步骤1—直觉: 用日常经验类比引入（"想象你在开车..."）
步骤2—形式化: 给出严格数学定义（Definition框）
步骤3—实例: 用一个最简单的例子验证定义
```

### 难度递进设计
```
Level 1: 概念理解题（"什么是IDZ模型的三个参数？"）
Level 2: 简单计算题（"给定以下渠池参数，求传递函数"）
Level 3: 分析题（"比较PID和MPC在以下工况下的控制效果"）
Level 4: 综合设计题（"为某灌区设计一套DMPC控制方案"）
```

### 教材vs专著的风格差异

| 维度 | 教材 | 专著 |
|------|------|------|
| 读者 | 研究生/高年级本科生 | 同行研究者 |
| 语气 | 引导式（"我们来看..."） | 陈述式（"本节给出..."） |
| 数学 | 每步推导+物理解释 | 关键步骤+参考文献 |
| 习题 | 必须有，分层设计 | 可选 |
| 案例 | 简化的教学案例 | 真实工程案例 |

## 技术报告写作技法

### 执行摘要写法（独立可读，1页以内）
```
段1: 项目背景和目标（为什么做这个项目）
段2: 主要技术路线（怎么做的，一句话概括）
段3: 核心成果（3-5个定量结论）
段4: 关键建议（按优先级排序，每条有行动主体）
```

### 数据呈现四原则
1. **来源透明**: 每个数据标注来源和采集时间
2. **口径一致**: 同一指标前后使用同一统计口径
3. **趋势可见**: 时间序列数据至少3个时间点
4. **对比明确**: 对比组的条件差异明确列出

### 结论与建议的可操作性
```
❌ "建议加强系统建设"（废话）
✅ "建议在2026年Q3前完成XXX系统升级，预算约120万元，
    责任单位为信息中心，关键风险为接口协议兼容性"
```

## 技术标准写作技法 (STD-CN/STD-INT)

### 条文撰写黄金规则

**一条一事**: 每个条文只规定一件事。如果一个条文需要"和""或"连接多个要求，应拆分。

**可测试性**: 每个"应"(shall)条文都要能回答：「怎么检验这个要求是否满足？」

**条文层次**：
```
5 一般规定                    ← 章标题
5.1 系统架构                  ← 节标题
5.1.1 系统应采用分布式架构... ← 条文(最小可执行单元)
```

### 条文用语精确化

| 意图 | 国内(GB/T 1.1) | 国际(ISO) | 错误用法 |
|------|---------------|-----------|---------|
| 强制要求 | 应 | shall | 必须/需要/要求 |
| 强制禁止 | 不应 | shall not | 不可以/禁止/不许 |
| 推荐 | 宜 | should | 建议/最好/推荐 |
| 允许 | 可 | may | 能够/可以(口语) |
| 陈述事实 | (直接陈述) | (直接陈述) | 应该是/一般为 |

### 术语定义格式（属概念+种差）
```
3.1 水网自主运行等级 water network autonomy level; WNAL
   反映水网系统在无人干预条件下自主完成调控任务的能力等级(3.2)的分级体系。
   注: 分为L0至L5共六个等级，参照SAE J3016自动驾驶分级思路。
```

## 演示文稿(PPT)叙事技法

### 三种叙事框架

**框架A: 问题→方案→验证（学术汇报/项目答辩）**
```
Act 1 (20%): 问题有多严重？（数据+痛点+紧迫感）
Act 2 (50%): 我们的方案（理论→方法→系统→架构）
Act 3 (30%): 证据（工程数据→对比→效果）
```

**框架B: 故事→洞察→行动（产品发布/领导汇报）**
```
Act 1 (15%): 一个具体的故事/场景（调度员凌晨3点值班...）
Act 2 (25%): 这背后的深层问题（为什么水网不能像电网一样自动？）
Act 3 (40%): 我们的答案（HydroOS演示+数据）
Act 4 (20%): 下一步行动（路线图+合作邀请）
```

**框架C: 对标→差距→路径（国际会议）**
```
Act 1 (20%): 自动驾驶/电力系统已经做到的（建立类比锚点）
Act 2 (20%): 水利系统的差距（为什么水利落后10年？）
Act 3 (40%): CHS框架如何填补差距（WNAL对标SAE J3016）
Act 4 (20%): 国际合作机会（IAHR WG + ISO标准）
```

### 每页设计原则

**信息层次**: 标题(核心结论) → 图/表(证据) → 标注(解释) → Notes(讲稿)
- 标题不是"研究方法"，而是"IDZ模型将计算时间缩短95%"
- 图表自己能讲故事，不依赖讲者口述

**视觉节奏**:
```
高密度页(图表+数据) → 高密度页 → 呼吸页(一句话+大图) → 高密度页 → 过渡页
```

### Speaker Notes写法
```
[转场] "在了解了理论框架之后，我们来看它在实际工程中的表现。"
[核心] "这张图展示了南水北调中线某渠池的水位控制效果。
        蓝线是传统PID，红线是我们的IDZ-MPC方案。
        可以看到，水位偏差从±15cm降到了±3cm。"
[停顿] "（停顿2秒，让数据沉淀）"
[过渡] "接下来我们看看在更极端的工况下..."
[时间] "（本页约1.5分钟，累计12分钟）"
```

## 参考文献写作技法（所有含引用的文体通用）

### 铁律：绝不编造

参考文献编造是最严重的学术不端之一。每条文献必须：
- 真实存在（用web_search验证）
- 作者、年份、期刊、卷期、页码全部正确
- 不确定时标注 `⚠️ 待核实`，由用户人工确认

### 文献选择策略

**漏斗式选文献**：
```
第1层: 该领域的经典/奠基性文献（必引，如Wylie 1969, Malaterre 1998）
第2层: 近10年的综述论文（帮助读者快速了解领域全貌）
第3层: 与本文方法直接相关的近5年论文（体现本文的创新增量）
第4层: 本文引用数据/模型/算法的原始出处（确保可追溯性）
第5层: 自引（Lei 2025a-d等，控制在合理比例内）
```

**选文献的三个"不"**：
- 不选只因为它增加引用数量（每条文献要有存在的理由）
- 不选无法获取原文的文献（至少能读到摘要）
- 不选与本文论点矛盾但未在正文中讨论的文献

### 正文中的引用技法

**引用三种姿势**：
```
姿势1（背景式）: "Recent studies have demonstrated... (Lei et al., 2025a; Wang & Zhang, 2024)."
  → 用于综述段落，多个文献佐证一个观点

姿势2（归属式）: "Lei et al. (2025a) proposed the CHS framework, which..."
  → 用于特定方法/观点的归属，强调某人的贡献

姿势3（对比式）: "Unlike the PID-based approach in [3], our method..."
  → 用于与前人工作的对比，突出差异
```

**引用密度节奏**：
```
Introduction: 最密集（几乎每句都有引用，建立学术语境）
Methodology: 中等（引用方法来源和数学基础）
Results: 最低（主要展示自己的结果）
Discussion: 再次升高（与前人结果对比）
```

### 书稿/教材的参考文献组织

**章末文献分类法**（适用于大型教材）：
```
参考文献

基础理论:
[1] Chaudhry M H. Open-Channel Hydraulics[M]. 3rd ed. Springer, 2022.
[2] Åström K J, Murray R M. Feedback Systems[M]. 2nd ed. Princeton, 2021.

核心方法:
[3] Litrico X, Fromion V. Modeling and Control of Hydrosystems[M]. Springer, 2009.
[4] Lei X H, et al. ...[J]. WRR, 2025.

工程应用:
[5] 南水北调中线建管局. ...中线工程运行管理手册[M]. 2023.
```

> 注意：分类法仅为读者便利，正式出版物中仍需按编号或字母统一排列。

### 自引的艺术

**自引的正当理由**（必须满足至少一条）：
- 本文是前文的直接延续（如"在Lei 2025a的基础上..."）
- 前文提供了本文使用的方法/数据/框架
- 前文是该领域为数不多的相关文献之一

**自引的红线**：
- 自引不是为了提高引用数，而是为了学术诚信和可追溯性
- SCI自引率控制在15-25%，中文10-20%，书稿≤15%
- 不在不相关的上下文中强行自引
