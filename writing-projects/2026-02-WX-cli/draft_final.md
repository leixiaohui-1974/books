# 为什么 AI 大厂都在做命令行版本

2025 年底到 2026 年初，一件反直觉的事情正在发生。Anthropic 推出了 Claude Code，Google 推出了 Gemini CLI，OpenAI 推出了 Codex CLI。三家最大的 AI 公司，不约而同地做了一个"复古"的产品——命令行工具。

在图形界面统治了四十年之后，黑底白字的终端窗口正在成为 AI 应用的新主场。

---

## 网页版的天花板

打开 ChatGPT 或 Claude 的网页版，体验很好：输入问题，得到回答，可以聊天，可以上传文件。但用得越多，你越会发现一个根本性的限制——它看不到你的工作现场。

你复制一段代码到聊天框里请 AI 帮你改。AI 改好了，你再复制回去。但它不知道这段代码在哪个文件、依赖了哪些库、和其他模块是什么关系。这就像你拍了一张照片发给远方的医生，让他远程诊断，他能给建议，但他看不到你的全部病历。

网页版本质上是一个"隔离沙箱"。你和 AI 之间隔着一个浏览器，一个复制粘贴。

---

## 命令行打破了这堵墙

Claude Code 的安装只需一行命令，启动后它直接运行在你的项目目录里。它能读你的文件，能看你的目录结构，能执行你的测试脚本，能提交你的代码。

这不是"你把问题发给 AI"，而是"AI 来到你的工作现场"。

差别有多大？举个真实场景。在网页版里，你说"帮我改一下登录模块的报错处理"，AI 会给你一段通用代码。

在 Claude Code 里，你说同样的话，它先读你的项目结构，找到登录模块，看看你现有的报错处理逻辑。然后检查框架版本，直接在原文件上修改，改完还帮你跑测试确认没有破坏别的功能。

一句话的区别：网页版回答问题，命令行版解决问题。

---

## 三家的不同赌注

虽然都是命令行工具，三家的设计哲学截然不同。

Anthropic 的 Claude Code 押注在推理深度上。它会在动手之前先"想清楚"，支持多文件协同编辑，修改完能自我检查和纠错。在 SWE-bench 真实代码修复测试中，Claude 的成绩超过 80%，大幅领先另外两家。

适合复杂项目的深度重构。

Google 的 Gemini CLI 押注在规模和免费上。百万级 token 的上下文窗口，意味着它能一次性"看到"整个中大型代码仓库。每天免费 1000 次请求，是目前入门门槛最低的选择。

适合大项目的快速浏览和重构。

OpenAI 的 Codex CLI 押注在开源和安全上。完全开源，沙箱隔离执行，权限控制细致。适合对安全性要求高的团队和企业环境。

三种哲学，三种入口，同一个方向。

---

## 不只是写代码

命令行 AI 的意义远不止于编程。

我自己用 Claude Code 做的最多的事情其实不是写代码，而是管理一个 25 万字的学术书稿项目。三个 GitHub 仓库、几百个文件、十几章交叉引用——在网页版里根本没法操作。

但在 Claude Code 里，我说"检查第八章和第五章的术语是否一致"，它就真的去翻两个章节的文件，逐条对照，找出不一致的地方。

这件事的本质是什么？是 AI 从"对话伙伴"变成了"工作搭档"。对话伙伴只能聊天，工作搭档能碰你的文件、跑你的脚本、查你的数据。

Anthropic 今年还推出了 Cowork，一个桌面版本，面向非程序员。它的底层逻辑和 Claude Code 一模一样：给 AI 一个虚拟机环境，让它直接操作文件和工具。只不过把命令行界面换成了图形界面。

从网页聊天到命令行工具再到桌面代理，演进方向非常清晰：让 AI 离你的工作越来越近。

---

## 为什么是现在

命令行 AI 之所以在 2025 年集中爆发，不是巧合，而是几个条件同时成熟。

第一是模型能力。以前的模型连写一段完整函数都费劲，现在能处理整个代码仓库级别的上下文。Gemini 的百万级 token 窗口，Claude 的 20 万 token 精准窗口，都足以支撑复杂项目的理解和修改。

第二是工具协议。Anthropic 的 MCP 和 Google 的 A2A 协议在 2025 年相继发布，成为连接 AI 和外部工具的标准化接口。有了这些协议，命令行 AI 才能规范地调用工具、读写文件、执行命令。

第三是信任基础。经过两年多的网页版使用，开发者已经建立了对大模型能力的基本信任。现在的问题不是"AI 能不能帮我"，而是"怎么让它帮得更高效"。命令行就是那个更高效的方式。

## 这跟水利有什么关系

你可能觉得命令行 AI 是程序员的事。但把"代码仓库"换成"水利工程数据"，把"读文件、改代码、跑测试"换成"读监测数据、调仿真模型、跑调度方案"，逻辑完全一样。

水利行业的仿真、控制、调度系统，核心也是一堆文件、一堆脚本、一堆模型。现在调度员要用这些工具，得打开好几个软件、手动输入参数、等结果、再手动比选方案。

如果有一个类似 Claude Code 的命令行工具，调度员直接说"用昨天的降雨数据跑三套方案，比较哪个安全裕度最高"。系统自动读数据、调模型、出报告。

这不是科幻，而是 Claude Code 正在编程领域做的事情。水利行业需要的只是把底层的仿真和控制工具接进去。

---

## 给你的一点建议

如果你还没用过命令行版 AI，可以从 Gemini CLI 开始，免费，门槛最低。如果你已经有编程经验或者需要处理复杂项目，Claude Code 是目前综合能力最强的选择。

但不管选哪个，核心体会是一样的：一旦你用过"AI 在你的工作现场"的感觉，就再也回不去"复制粘贴到聊天框"的模式了。

命令行看上去是退了一步，实际上是进了一大步。
