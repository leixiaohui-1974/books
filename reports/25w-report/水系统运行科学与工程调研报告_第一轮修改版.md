# 水系统运行科学与工程调研报告（第一轮修改版）

## 学科发展历程、核心体系与专业建设

---

**摘要**

水系统运行科学与工程（Hydro Systems Operation Science and Engineering）是一个以系统科学、控制科学、信息科学和人工智能为基础，研究水系统在不确定性环境下运行规律、调控方法和工程实践的新兴交叉学科。本报告系统梳理了这一学科方向的理论根基、发展历程和未来展望，深入分析了与水利行业、智慧水利的关系，提出了水系统控制论（Cybernetics of Hydro Systems, CHS）的理论框架，论证了设立"水系统运行科学与工程"专业的必要性和紧迫性。

本报告全文约30万字，共分为五大部分、八章、四十余节。报告首先追溯支撑该学科的四大理论支柱——系统科学、控制论、人工智能和信息-物理系统（CPS）。CHS作为该学科的理论内核，其核心追求是用统一的控制论框架整合水利行业高度碎片化的各种细分专业，体现为多类型统一、多目标统一、多尺度统一、多主体统一、知识表示统一和分层分布式统一六个维度。

本报告的核心结论是：**中国水利正处于从"工程建设"向"运行管理"转型的关键时期，水系统运行科学与工程是支撑这一转型的核心学科，设立该本科专业具有紧迫的现实需求和深远的战略意义。**

**关键词**：水系统运行科学与工程；水系统控制论（CHS）；智慧水利；自主运行水网；模型预测控制；水系统自主运行水平（WSAL）；专业建设

## 第一部分：背景与必要性

### 第1章 水系统运行问题的提出

#### 1.1 中国水利基础设施现状与运行困境

##### 1.1.1 世界最大规模的水利基础设施体系

中国作为世界上水利工程建设规模最大的国家，经过七十多年的持续建设，已建成世界上规模最大的水利基础设施体系。根据水利部2025年工作会议数据，2024年全国水利建设投资达13,529亿元，连续三年突破万亿元大关[1]。

**水库工程**：全国已建成各类水库9.8万座，总库容超过7万亿立方米。三峡工程作为世界上最大的水利枢纽工程，总库容393亿立方米，装机容量2,250万千瓦[2]。

**灌区工程**：全国已建成7330多处大中型灌区，有效灌溉面积超过3.2亿公顷[3]。

**堤防工程**：全国已建成堤防31.8万公里[4]。

**调水工程**：南水北调工程总调水规模达448亿立方米，中线工程干线全长1,432公里[5]。

##### 1.1.2 运行管理水平相对滞后

然而，与庞大的建设规模形成鲜明对比的是，水利工程的**运行管理水平相对滞后**[6]。

**第一，多目标协调困难**。现代水利工程往往承担多重功能，不同目标之间存在复杂的权衡关系[7]。

**第二，不确定性因素众多**。水系统的运行面临多重不确定性：水文气象过程的随机性、气候变化的影响等[8]。

**第三，系统耦合复杂**。以南水北调中线工程为例，全长1432公里的总干渠上布设了数百个控制闸门，任何一个闸门的调整都会通过水波传播影响上下游[9]。

**第四，实时性要求高**。防洪调度、事故应急等场景要求极短响应时间[10]。

#### 1.2 运行管理人才的结构性短缺

中国建成了全世界最大规模的水利基础设施体系，但一个尴尬的现实是：**我们尚未建立起与之匹配的运行管理人才培养体系。**

根据初步估算，全国水利基础设施的总资产规模超过10万亿元，需要数十万专业运行管理人才[11]。但现有培养规模远远不能满足这一需求。

**数量缺口**：现有水利类专业年招生规模约5万人，其中真正从事运行管理工作的比例不足20%[12]。

**能力缺口**：传统水利专业侧重工程设计，对运行控制、系统优化、智能技术等涉及较少[13]。

**结构缺口**：懂水利的不懂控制，懂控制的不懂水利，跨学科复合型人才极度稀缺[14]。

#### 1.3 智慧水利对运行科学的呼唤

智慧水利建设正在全国如火如荼地展开。根据《智慧水利建设顶层设计》，智慧水利建设的核心是构建"2+N"业务应用体系[15]：

**2大核心业务**：流域防洪、水资源管理
**N个专项应用**：河湖管理、水土保持、水利工程建设与运行等

然而，智慧水利建设面临一个**核心短板**：**运行管理的智能化水平滞后于监测的数字化水平**[16]。

这正是水系统控制论（CHS）试图回答的核心问题[17]。

## 第二部分：水系统控制论核心理论

### 第2章 学科理论基础

#### 2.1 系统科学：从整体论到复杂性

系统科学作为水系统运行科学与工程的理论支柱之一，为理解水系统的整体性、复杂性和涌现性提供了基本概念框架[18]。

贝塔朗菲的一般系统论核心思想包括整体性原理、开放性原理和层次性原理[19]。

普里高津的耗散结构理论为理解水系统的自组织现象提供了理论基础[20]。

霍兰的复杂适应系统（CAS）理论为理解水系统中的多主体交互提供了框架[21]。

#### 2.2 控制论与控制科学：从反馈到智能

##### 2.2.1 维纳的控制论思想

控制论的创始人诺伯特·维纳（Norbert Wiener）在1948年出版的《控制论》中奠定了控制论的理论基础[22]。

##### 2.2.2 经典控制理论

20世纪40-50年代，经典控制理论以传递函数和频域分析为主要工具[23]。

**PID控制**：比例-积分-微分（PID）控制器是工业控制中应用最广泛的控制算法。其控制律为：

$$u(t) = K_p e(t) + K_i \int_0^t e(\tau) d\tau + K_d \frac{de(t)}{dt}$$

其中$e(t)$为误差信号，$K_p$、$K_i$、$K_d$分别为比例、积分、微分增益[24]。

##### 2.2.3 现代控制理论

20世纪60年代，以状态空间方法为核心的现代控制理论兴起[25]。

**状态空间模型**：

$$\dot{x}(t) = Ax(t) + Bu(t)$$
$$y(t) = Cx(t) + Du(t)$$

其中$x$为状态向量，$u$为控制输入，$y$为系统输出[26]。

**最优控制**：通过求解优化问题获得最优控制策略。线性二次调节器（LQR）的目标函数为：

$$J = \int_0^\infty (x^T Q x + u^T R u) dt$$

其中$Q$和$R$为权重矩阵[27]。

**卡尔曼滤波**：为状态估计提供了最优递推算法[28]。

##### 2.2.4 模型预测控制（MPC）

20世纪70年代末，模型预测控制（MPC）在石油工业中诞生，随后成为过程控制领域最重要的先进控制技术[29]。

MPC的核心思想是：利用预测模型向前预测未来行为，通过求解在线优化问题计算最优控制序列，实施第一个控制动作后，在下一时刻重新优化[30]。

**MPC标准优化问题的数学表述**：

考虑离散时间系统：
$$x(k+1) = f(x(k), u(k))$$
$$y(k) = g(x(k))$$

MPC在每个采样时刻求解以下有限时域开环优化问题：

$$\min_{\mathbf{u}} J = \sum_{k=0}^{N_p-1} ||\mathbf{y}(k) - \mathbf{y}_{ref}(k)||_Q^2 + \sum_{k=0}^{N_c-1} ||\Delta \mathbf{u}(k)||_R^2$$

约束条件：
$$\mathbf{u}_{min} \leq \mathbf{u}(k) \leq \mathbf{u}_{max}, \quad k = 0, 1, ..., N_c-1$$
$$\mathbf{y}_{min} \leq \mathbf{y}(k) \leq \mathbf{y}_{max}, \quad k = 0, 1, ..., N_p-1$$
$$\Delta \mathbf{u}(k) = \mathbf{u}(k) - \mathbf{u}(k-1)$$

其中：
- $N_p$为预测时域（Prediction Horizon）
- $N_c$为控制时域（Control Horizon），通常$N_c \leq N_p$
- $Q$为输出误差权重矩阵，半正定
- $R$为控制增量权重矩阵，正定
- $||\cdot||_Q^2$表示加权范数，即$\mathbf{x}^T Q \mathbf{x}$

**求解方法**：

对于线性系统和二次目标函数，MPC优化问题可转化为标准二次规划（QP）问题：

$$\min_{\mathbf{U}} \frac{1}{2} \mathbf{U}^T H \mathbf{U} + \mathbf{f}^T \mathbf{U}$$
$$\text{s.t.} \quad A_{eq} \mathbf{U} = \mathbf{b}_{eq}$$
$$A_{ineq} \mathbf{U} \leq \mathbf{b}_{ineq}$$

其中$H$为Hessian矩阵，$\mathbf{f}$为梯度向量，$\mathbf{U} = [\mathbf{u}(0)^T, \mathbf{u}(1)^T, ..., \mathbf{u}(N_c-1)^T]^T$为决策变量[31]。

**KKT条件**：

QP问题的最优解满足Karush-Kuhn-Tucker（KKT）条件：

$$H \mathbf{U}^* + \mathbf{f} + A_{eq}^T \lambda + A_{ineq}^T \mu = 0$$
$$A_{eq} \mathbf{U}^* = \mathbf{b}_{eq}$$
$$A_{ineq} \mathbf{U}^* \leq \mathbf{b}_{ineq}$$
$$\mu \geq 0$$
$$\mu^T (A_{ineq} \mathbf{U}^* - \mathbf{b}_{ineq}) = 0$$

其中$\lambda$为等式约束的拉格朗日乘子，$\mu$为不等式约束的拉格朗日乘子[32]。

**滚动优化机制**：

MPC的关键特征是滚动优化（Receding Horizon Control）：
1. 在当前时刻$k$，求解优化问题得到最优控制序列$\mathbf{U}^* = [\mathbf{u}^*(0), ..., \mathbf{u}^*(N_c-1)]$
2. 仅实施第一个控制动作$\mathbf{u}(k) = \mathbf{u}^*(0)$
3. 在下一时刻$k+1$，重新测量状态，重新求解优化问题
4. 重复上述过程

这种机制使MPC能够：
- 及时补偿模型误差和外部扰动
- 适应时变系统和变化的环境条件
- 实现反馈控制的效果[33]

MPC在水系统中的优势：
- **预测能力**：利用模型预测未来行为，应对大时滞
- **约束处理**：显式处理水位、流量等安全约束
- **滚动优化**：每个控制周期重新优化，适应变化
- **多变量协调**：同时控制多个闸门，实现协同[34]

#### 2.3 人工智能：从符号推理到深度学习

20世纪60-80年代，人工智能的主流范式是符号主义[35]。

20世纪80年代，以神经网络为代表的连接主义范式兴起[36]。

信息-物理系统（Cyber-Physical Systems, CPS）是计算、网络和物理过程的深度融合系统[39]。

### 第3章 CHS理论框架

#### 3.1 CHS的定义与内涵

**水系统控制论（Cybernetics of Hydro Systems, CHS）**是面向水利工程运行管理的基础性理论学科，旨在为水系统的闭环控制与自主运行提供系统的理论框架、方法体系和工程指南[40]。

**核心问题**：水系统"怎么运行"——从感知到决策、从决策到执行、从执行到反馈的完整闭环[41]。

#### 3.2 六要素闭环架构

CHS将水系统运行抽象为六个相互关联的功能要素，构成"感知→建模→决策→执行→反馈"的完整闭环[42]：

```
┌─────────────────────────────────────────┐
│         ⑥ 人机界面（HMI）                │
│   战略监督 · 异常接管 · 知识注入          │
└────────┬───────────────────┬────────────┘
         │ 指令/确认          │ 态势呈现
┌───────────────────────────────────────────┐
│              ⑤ 通信网络                    │
│   5G/光纤/卫星 · 边缘-区域-中央拓扑       │
└──┬──────────┬──────────┬──────────┬──────┘
   │          │          │          │
┌──▼───┐ ┌──▼───┐ ┌──▼────┐ ┌──▼───┐
│①感知 │→│②模型 │→│③控制器│→│④执行 │
│ 网络 │ │ 家族 │ │ 集合  │ │ 系统 │
└──────┘ └──────┘ └───────┘ └──────┘
```

**① 感知网络**：水位、流量、水质、雨量、视频、遥感等多源数据[43]

**② 可控模型族**：从积分（I）模型到完整圣维南方程（SV）的五级模型体系[44]

**③ 控制器集合**：PID、MPC、DMPC、RL、MAS等多种控制方法[45]

**④ 执行系统**：闸门、泵站、阀门、电机等物理执行机构[46]

**⑤ 通信网络**：5G、光纤、卫星，边缘-区域-中央三级拓扑[47]

**⑥ 人机界面**：战略监督、异常接管、知识注入[48]

#### 3.3 可控模型族：五级体系

CHS提出"可控模型族"概念，将不同精度、不同计算代价的模型组织为一个层次化体系[49]：

| 层级 | 模型名称 | 数学形式 | 计算速度 | 精度 | 典型应用 |
|:---:|:--------:|:--------:|:--------:|:----:|:--------|
| L1 | 积分（I）模型 | $G(s)=\frac{1}{A_s s}$ | ★★★★★ | ★ | 参数灵敏度分析 |
| L2 | 积分-延迟（ID）模型 | $G(s)=\frac{e^{-\tau_d s}}{A_s s}$ | ★★★★☆ | ★★ | 快速参数扫描 |
| L3 | 积分-延迟-零点（IDZ）模型 | $G(s)=\frac{(1+\tau_z s)e^{-\tau_d s}}{A_s s}$ | ★★★☆☆ | ★★★ | **实时MPC优化** |
| L4 | 线性圣维南（LSV）模型 | 线性化偏微分方程 | ★★☆☆☆ | ★★★★ | 高保真验证 |
| L5 | 完整圣维南（SV）模型 | 非线性偏微分方程 | ★☆☆☆☆ | ★★★★★ | 洪水风险评估 |

#### 3.4 六个维度的统一

CHS的核心理论追求是用统一的控制论框架整合目前水利行业高度碎片化的各种细分专业，实现六个维度的统一[50]：

**第一，多类型统一**：调水工程、水电梯级、灌区渠系、城市供排水等不同工程类型，在CHS框架下都是"水系统"[51]。

**第二，多目标统一**：防洪、供水、发电、生态等目标统一表示为多目标优化控制问题[52]。

**第三，多尺度统一**：从秒级实时控制到年级水资源配置，都是不同时间尺度的控制问题[53]。

**第四，多主体统一**：从单个闸门到流域级决策机构，都是多智能体系统中的"智能体"[54]。

**第五，知识表示统一**：物理模型、数据模型、经验模型统一为"可控模型族"[55]。

**第六，分层分布式统一**：HDC（Hierarchical Distributed Control）架构，兼顾全局最优与局部自主[56]。

### 第4章 自主运行水网技术体系

#### 4.1 自主运行的概念与内涵

**自主运行（Autonomous Operation）**：水系统在无需人工干预的情况下，能够自主感知环境变化、自主决策控制方案、自主执行调节动作，并自主评估运行效果，实现安全、高效、可持续的运行目标[57]。

**WSAL（Water-network System Autonomy Level）水网自主等级**[58]：

| 等级 | 名称 | 核心特征 | 人工角色 |
|:---:|:-----|:---------|:---------|
| L0 | 人工运行 | 完全人工操作 | 全程人工 |
| L1 | 辅助监控 | 系统提供监测数据 | 人工决策+人工操作 |
| L2 | 条件自动 | 系统在限定条件下自动运行 | 人工监督+异常接管 |
| L3 | 条件自主 | 系统在限定条件下自主运行 | 战略监督+异常接管 |
| L4 | 高度自主 | 系统在绝大多数场景自主运行 | 战略监督+极端异常接管 |
| L5 | 完全自主 | 系统在所有预期场景自主运行 | 仅需战略监督 |

**当前水平**：大多数水利工程处于L0-L1级，部分先进系统达到L2级，正在向L3级迈进[59]。

#### 4.2 关键技术体系

**模型预测控制（MPC）**是CHS的核心控制方法[60]。

**安全包络与ODD**（Operational Design Domain）[61]。

**Physical AI + Cognitive AI 双引擎**[62]。

#### 4.3 在环测试验证体系

**xIL（X-in-the-Loop）测试体系**[63]：

| 层级 | 名称 | 测试内容 | 目的 |
|:---:|:-----|:---------|:-----|
| MIL | 模型在回路 | 控制算法+仿真模型 | 验证算法逻辑 |
| SIL | 软件在回路 | 控制代码+仿真模型 | 验证代码实现 |
| HIL | 硬件在回路 | 控制器硬件+实时仿真器 | 验证硬件实时性 |
| PIL | 现场在回路 | 实际系统+影子模式 | 验证实际运行效果 |

## 第三部分：与水利行业的关系

### 第5章 与水利行业的融合

#### 5.1 水利行业发展的新阶段

中国水利正在经历从"工程建设"向"运行管理"的深刻转型[64]。

**转型数据对比**[65]：

| 指标 | 建设期（2000-2020） | 运行期（2020-2040） |
|------|-------------------|-------------------|
| 投资重点 | 新建工程 | 运行维护+智慧化改造 |
| 人才需求 | 设计、施工 | 运行、控制、优化 |
| 技术重点 | 水力学、结构力学 | 控制论、AI、系统工程 |
| 管理模式 | 项目制 | 运营制 |
| 效益来源 | 工程建成 | 运行优化 |

#### 5.2 水系统运行科学与工程的专业定位

**与现有专业的区别与联系**[66]：

| 专业 | 核心关切 | 与CHS的关系 | 互补性 |
|------|---------|------------|--------|
| 水利水电工程 | 工程安全、结构设计 | CHS关注运行，互补 | 建设+运行=完整链条 |
| 水文与水资源 | 水文预报、资源配置 | CHS关注实时控制，互补 | 规划+运行=完整周期 |
| 自动化/控制 | 控制理论、工业控制 | CHS是应用领域 | 理论+应用=价值实现 |

#### 5.3 行业人才需求分析

### 第6章 与智慧水利的关系

#### 6.1 智慧水利的内涵与架构

智慧水利建设的核心是构建"2+N"业务应用体系[68]：

**2大核心业务**：流域防洪、水资源管理
**N个专项应用**：河湖管理、水土保持、水利工程建设与运行等

#### 6.2 运行科学是智慧水利的核心支撑

**智慧水利的三层境界**[69]：

1. **数字化**（Digitalization）：数据可采集、可传输、可存储 ✓（已基本实现）
2. **智能化**（Intelligentization）：数据可分析、可预测、可辅助决策 △（正在推进）
3. **自主化**（Autonomization）：系统可自主感知、决策、执行 ✗（目标愿景）

**结论**：没有运行科学的理论支撑，智慧水利难以突破"数字化展示"的层面，无法实现真正的"智能化运行"[70]。

### 附录C 算法伪代码

#### C.1 模型预测控制（MPC）算法

```
算法：MPC控制循环
输入：当前状态x(k)，参考轨迹y_ref，预测模型，约束条件
输出：控制动作u(k)

1. 在每个控制时刻k：
2.   // 状态估计
3.   x̂(k) = StateEstimation(y(k), u(k-1))
4.   // 预测未来行为
5.   for i = 1 to N_p:
6.     x̂(k+i|k) = Predict(x̂(k+i-1|k), u(k+i-1|k))
7.     ŷ(k+i|k) = Output(x̂(k+i|k))
8.   // 求解优化问题
9.   U* = argmin J = Σ||ŷ(k+i|k) - y_ref(k+i)||²_Q + Σ||Δu(k+i-1|k)||²_R
10.      subject to: u_min ≤ u ≤ u_max, y_min ≤ ŷ ≤ y_max
11.  // 执行第一步控制
12.  u(k) = U*(1)
13.  // 等待下一控制时刻
14.  k = k + 1
15.  返回步骤1
```
