## 📝 各案例修改要点

### 6.2 点状系统：沙坪水电站（定性）

**原始报告关键信息**（2020 年）：
- 河床式径流电站
- 调节库容较小（日调节）
- 多台灯泡贯流式机组
- 下游有城区防洪任务
- 死水位 550m，正常蓄水位 554m（**不写入书稿**）
- 入库流量 500-4500 m³/s（**不写入书稿**）

**书稿定性表述**：
```
某大型水电站位于我国西南某河流中游，是干流梯级水电开发规划中的一级电站。
电站为河床式径流电站，水库调节库容较小，调节性能为日调节。

控制挑战：
1. 发电约束：调节库容小，调峰能力有限，机组负荷调整速率受限
2. 防洪约束：下游有城区防洪保护任务，需预泄腾库
3. 生态约束：下泄生态流量需满足枯水期和丰水期的不同要求
4. 核心矛盾：发电（高水位）、防洪（低水位）、生态（稳定下泄）三者冲突

ODD 定义（定性）：
- 常规工况：入库流量在设计范围内，库水位在正常蓄水位附近
- 异常工况：入库流量接近设计上限/下限，库水位接近汛限水位/死水位
- 极端工况：入库流量接近防洪极限，库水位接近坝顶高程

对应 CHS 原理：
- 传递函数化：将水库抽象为输入（入库流量）- 输出（出库流量）传递函数
- 安全包络：绿/黄/红三区划分，对应不同控制策略
```

### 6.3 链状系统：梯级协调（定性）

**原始报告关键信息**：
- 瀑布沟、深溪沟、枕头坝三站
- 瀑布沟库容大，深溪沟和枕头坝库容小
- 站间水流时滞约 90 分钟和 40 分钟
- 三站共 35 台机组
- 联合躲避振动区

**书稿定性表述**：
```
某梯级水电站群由三座电站自上而下紧密相连，构成一个典型的"龙头水库 + 反调节电站 + 二重反调节电站"的梯级组合。

控制挑战：
1. 调节不对称：上游水库库容大，下游水库库容小，对上游调度动作极度敏感
2. 水力耦合：出库流量经一定时滞传播至下游，时滞随流量变化
3. 电力耦合：三站共同承担电网负荷指令，需协调分担
4. 振动区约束：各机组有振动区，需联合躲避

ODD 定义（定性）：
- 单站 ODD：各站独立的水位、流量、负荷约束
- 梯级 ODD：多站 ODD 的交集，考虑耦合约束
- 关键问题：上游水位变化对下游的约束传播

对应 CHS 原理：
- 分层分布式：电网 - 集控 - 电站三层架构
- ODD 定义：多站 ODD 的交集与级联约束传播
- 安全包络：梯级联合安全包络
```

### 6.4 网状系统：长距离调水（定性）

**胶东原始资料关键信息**：
- 全长约 571 km
- 13 级泵站、95 个闸站、46 个阀站
- 多水源（黄河、长江、当地水）
- 多目标（工业、农业、生活、生态）
- 多工况（冰期、汛期、检修）

**书稿定性表述**：
```
某长距离调水工程全长数百公里，串联多个泵站、闸站和调蓄水库，形成复杂的水网系统。

控制挑战：
1. 多水源联合：不同水源水质差异大，需动态追踪
2. 多工程混联：明渠、暗涵、有压管道交替，水流形态频繁切换
3. 多目标供水：工业、农业、生活用水需求不同，存在资源争夺
4. 多工况切换：冰期、汛期、检修等工况边界跳跃大
5. 多层级管理：多个分中心、管理站之间的调度协同

ODD 定义（定性）：
- 多态 ODD：不同工况下 ODD 边界动态变化
- 冰期工况：ODD 边界收缩
- 汛期工况：预留防洪库容

对应 CHS 原理：
- 认知增强：大语言模型提供决策支持
- 人机共融：云边协同，边缘本地控制 + 云端全局优化
- 在环验证：MIL/SIL/HIL三级验证体系
```

### 6.5 国内外典型案例对比（新增）

**国际案例（文献支撑）**：

| 案例 | 国家 | 类型 | 文献来源 | 可引用点 |
|------|------|------|---------|---------|
| 中央亚利桑那调水工程 (CAP) | 美国 | 长距离调水 | ASCE MOP 131 (2014) | MPC 应用、分层控制 |
| Canal de Provence 灌区 | 法国 | 灌溉渠系 | Litrico & Fromion (2009) | 传递函数建模、频域控制 |
| 加州中央谷地引水工程 | 美国 | 跨流域调水 | US Bureau of Reclamation | 多目标优化 |

**书稿表述**：
```
6.5.1 美国中央亚利桑那调水工程（CAP）

CAP 工程全长 540 km，年调水量约 20 亿 m³，采用分层分布式控制架构。
其控制策略与本书第六章所述某调水工程有相似之处：
- 均采用 MPC 进行滚动优化
- 均定义了工况边界（类似 ODD）
- 均采用云边协同架构

不同之处：
- CAP 建设于 1990 年代，自动化水平相当于 WNAL L2
- 某调水工程建设于 2020 年代，智能化水平接近 WNAL L3

6.5.2 法国 Canal de Provence 灌区

Canal de Provence 灌区采用传递函数建模和频域控制方法，
与本书第二章所述水系统形式化方法一致：
- 将渠道抽象为传递函数
- 采用频域方法设计控制器
- 考虑时滞和不确定性

启示：CHS 理论的传递函数化原理在国际上已有成功应用。
```
