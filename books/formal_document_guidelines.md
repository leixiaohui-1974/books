# 正式文档写作规范指南

## 文档类型分类

### 1. 正式学术/技术文档（必须遵循严格规范）
- **学术论文**: 期刊论文、会议论文、学位论文
- **技术报告**: 研究报告、实验报告、项目报告  
- **专利文档**: 专利申请书、技术交底书、权利要求书
- **标准文档**: 技术标准、行业规范、企业标准
- **项目文档**: 可行性研究报告、实施方案、验收报告
- **官方文件**: 政府报告、政策建议、白皮书

### 2. 非正式文档（可适当灵活）
- **公众号文章**: 科普文章、技术分享、行业动态
- **内部备忘录**: 团队沟通、临时记录
- **个人笔记**: 学习笔记、思考记录

## 正式文档核心要求

### 1. 语言风格
- **专业性**: 使用专业术语，避免口语化表达
- **准确性**: 用词精确，避免模糊表述
- **客观性**: 基于事实和数据，避免主观臆断
- **一致性**: 术语、格式、风格保持统一

### 2. 结构规范
- **标准结构**: 遵循相应文档类型的国际/国家标准
- **逻辑清晰**: 论证严密，层次分明
- **引用规范**: 严格按照指定格式引用参考文献

### 3. 格式要求
- **排版规范**: 字体、字号、行距、页边距等符合要求
- **图表规范**: 图表编号、标题、说明完整规范
- **公式规范**: 数学公式编号、符号说明清晰

## 各类正式文档具体规范

### 学术论文
- **结构**: Abstract, Introduction, Methods, Results, Discussion, Conclusion, References
- **引用格式**: APA/MLA/Chicago/IEEE/Nature/Science 等按期刊要求
- **语言**: 学术英语或规范中文，被动语态适度使用

### 技术报告  
- **结构**: Executive Summary, Background, Methodology, Findings, Recommendations, Appendices
- **语言**: 专业、简洁、准确，避免营销语言
- **数据**: 提供完整的数据来源和分析方法

### 专利文档
- **结构**: Technical Field, Background Art, Summary, Detailed Description, Claims, Abstract
- **语言**: 法律技术语言，权利要求书用词精确无歧义
- **创新性**: 清晰描述技术方案的创新点和有益效果

### 标准文档
- **结构**: Scope, Normative References, Terms and Definitions, Technical Requirements, Test Methods
- **语言**: 规范性条款用"应"，资料性条款用"可"
- **一致性**: 与相关标准保持协调一致

## 质量检查清单

### 内容质量
- [ ] 专业术语使用准确
- [ ] 数据和事实准确无误  
- [ ] 逻辑论证严密完整
- [ ] 创新点和贡献明确
- [ ] 局限性和不足诚实说明

### 格式质量
- [ ] 结构符合文档类型要求
- [ ] 引用格式完全正确
- [ ] 图表编号连续完整
- [ ] 公式符号定义清晰
- [ ] 页眉页脚格式规范

### 语言质量
- [ ] 无语法错误和拼写错误
- [ ] 无口语化表达
- [ ] 术语使用一致
- [ ] 句子结构清晰
- [ ] 段落逻辑连贯

## 特别注意事项

1. **除公众号文章外，所有文档都必须遵循正式文档规范**
2. **学术诚信**: 严禁抄袭、数据造假、一稿多投
3. **版权合规**: 正确引用他人工作，获得必要授权
4. **保密要求**: 不泄露敏感信息和未公开数据
5. **审稿流程**: 重要文档需经过同行评审或专家审核

## AI辅助写作原则

1. **AI作为工具**: AI提供辅助，但最终责任在作者
2. **内容审核**: AI生成内容必须经过人工审核和修改
3. **原创性保证**: 确保最终文档具有足够的原创性
4. **专业判断**: AI建议需结合专业判断进行取舍
