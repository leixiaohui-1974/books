## 🔧 自动化检查项

1. **公式完整性检查**：
   - 行内公式：配对检查（$数量为偶数）
   - 公式块：配对检查（$$配对）
   - 编号连续性：(X-1)~(X-?)是否连续

2. **表格完整性检查**：
   - 表格数量统计
   - 表格标题与内容一致性
   - 行列对齐

3. **图片引用检查**：
   - 图片是否都使用GitHub raw链接
   - 图片是否实际存在
   - 编号一致性：fig_XX_YY

4. **参考文献检查**：
   - 编号连续性
   - 正文引用与列表匹配
   - DOI/URL格式

5. **交叉引用检查**：
   - 跨章节引用准确性
   - 图表引用准确性
   - 公式引用准确性

6. **术语一致性检查**：
   - 关键术语（CHS、WNAL、ODD、HDC等）在全书是否定义一致
   - 禁止别名清单（vs MEMORY.md）

7. **编号连续性检查**：
   - 章节号：Ch01~Ch15
   - 节号：§X.Y.Z是否连续
   - 表号：Table X-Y是否连续
   - 图号：fig_XX_YY命名规范

## 📊 评审深度设定

### P0 - 严格审查（必须通过）
- 逻辑循环论证、概念矛盾
- 核心定义与MEMORY.md不一致
- 参考文献虚构或无法验证
- 工程参数明显错误（超过±10%）

### P1 - 中度审查（应该通过）
- 表述不清晰或逗号问题
- 图表编号不连续
- 公式格式不规范
- 交叉引用错误

### P2 - 轻度审查（可优化）
- 表述冗长、需要精简
- 示例不够直观
- 排版美化
