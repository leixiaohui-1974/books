# 从 AI 助手到水网大脑：同一条进化路

2026 年 1 月，Anthropic 发布了桌面工具 Cowork。一个月内，软件行业 ETF 单日暴跌近 6%，汤森路透股价创下历史最大单日跌幅。华尔街的恐慌不是因为 AI 又聪明了一点，而是因为 AI 从"回答问题"变成了"完成工作"。

这条进化路径，水利人应该格外熟悉。

---

## 一个意外的平行

Anthropic 把自己产品的演进总结为三个阶段：Chat → Code → Cowork。Chat 是聊天，你问它答。Code 是写代码，AI 能读懂整个工程仓库并动手修改。Cowork 是干活，AI 在你电脑上开一个虚拟机，直接操作文件、调用工具、协调多个子任务。

这三步，恰好对应水网自主运行能力分级（WNAL）中的三个关键跃迁。

L1 是辅助信息——SCADA 采集数据，人看屏幕做判断，系统只管"汇报"。这就是 Chat 模式：你问什么它答什么，决策权完全在人。

L3 是条件自主——在设定好的运行设计域（ODD）内，系统能实现多目标协同优化控制。人只负责监督和处置异常。这像 Code 模式：给定明确边界，AI 自主完成复杂任务。

L5 是完全自主——全天候、全工况，系统自主运行并持续演进。这正是 Cowork 的野心：不再等你提问，而是主动分析、规划、执行、检查、迭代。

---

## 本质相同：从"回答"到"执行"

Chat 和 L1 的共同特征是什么？信息流是单向的。系统提供数据，人做决策。传感器告诉你水位是多少，Claude 告诉你答案是什么。有用，但被动。

Cowork 和 L3 以上的共同特征是什么？系统进入了"感知—认知—决策—控制"的闭环。Cowork 分析你的请求、制定计划、拆分子步骤、逐步执行、检查结果。水网自主控制系统监测水力状态、预测变化趋势、计算最优方案、下达控制指令、验证执行效果。

两者的架构惊人地相似。Cowork 在虚拟机沙箱里运行 agent 循环，水网在 HydroOS 操作系统里运行分层分布式控制。Cowork 通过连接器接入 Google Drive 和 Excel，水网通过标准化 API 接入传感器和执行器。

Cowork 需要用户授权才能删文件，水网在 ODD 边界外自动切入最小风险条件。

控制权的让渡，是最难的那一步。

---

## 慢，是有原因的

用过 Cowork 的人都知道：它比网页版慢很多。原因是每一步"思考—执行—检查"都是一次 API 调用，加上虚拟机的启动和文件读写开销。

水网控制系统也面临完全相同的挑战。从 L1 升级到 L3，系统响应链从"人看数据→人下命令"变成了"模型预测→方案优化→安全校验→指令下达→效果反馈"。环节多了，延迟也多了。

但这恰恰是可靠性的代价。Cowork 之所以在删除文件前弹出授权提示，是为了安全。水网之所以在 ODD 边界设置安全包络，也是为了安全。急不得。

---

## 插件与专业化：走向"你们公司的 AI"

2 月 25 日，Anthropic 把 Cowork 升级为企业级产品，推出了 HR、金融、工程等领域的插件模板。企业管理员可以搭建私有插件市场，把机构知识编码进 Claude。

产品负责人 Scott White 的原话很有意思：目标不是"通用的法律 AI"，而是"你们公司的法律 AI"。

这和水网领域正在发生的事一模一样。水系统控制论提出的 HydroOS 不是一个通用软件，而是一个开放平台。灌区的控制逻辑和输水干渠完全不同，城市供水管网的安全约束和梯级水电站也完全不同。HydroOS 通过标准化接口，让每个工程都能加载自己的模型、算法和控制策略——就像 Cowork 的插件。

通用平台加专业插件，这可能是复杂系统智能化的通用范式。

---

## 人没有被取代，角色在升级

华尔街的恐慌可以理解，但 Anthropic 自己的经济学家说了：目前没有看到大规模劳动力替代的证据。

水利行业也正在经历同样的讨论。L5 是"完全自主"，是不是意味着不需要人了？水系统控制论给出的答案很明确：不是。WNAL 体系中，即使是 L5，人的角色也是"监督"。从 L0 到 L5，人的角色从"操作者"变成了"监督者"再变成了"架构师"。

Cowork 的逻辑完全一致。它不是要替代知识工作者，而是把重复性的执行工作交给 AI，让人专注于判断、创意和战略。Kate Jensen 说得很清楚：工程师觉得 Claude Code 不可或缺，未来每个知识工作者对 Cowork 也会有同样的感受。

关键词不是"替代"，是"不可或缺"。

---

## 水利人应该关注什么

Cowork 的发展给水网智能化至少带来三个启示。

第一，agentic 架构已经成熟。Cowork 证明了"分析—规划—执行—检查"的 agent 循环在真实场景中是可行的。这正是水网从 L2 跃升到 L3 所需要的核心能力。

第二，连接器生态至关重要。Cowork 的价值不在于 Claude 本身变聪明了，而在于它能接入 Google Drive、Excel、DocuSign 这些工具。HydroOS 的价值也不在于算法本身，而在于它能统一接入传感器、执行器、水力模型和气象预报。

第三，安全框架先于功能扩展。Cowork 在每个敏感操作前都要求授权，企业管理员控制插件的访问权限。水网在每次自主决策前都要校验 ODD 边界。安全不是功能的对立面，而是前提。

---

## 同一条路，不同的河

AI 行业从 Chat 走到 Cowork，花了不到三年。水利行业从 L0 走到 L3，已经走了几十年，从 L3 到 L5 还有很长的路。

节奏不同，但方向相同：从被动的信息提供者，到主动的任务执行者，最终成为可信赖的自主系统。

下一次你打开 Cowork 看着它在虚拟机里一步步拆解任务的时候，不妨想想——这个过程，和大渡河梯级水电站群的联合优化调度，本质上是同一件事。
